DocketClock · technical architecture vision

Deadline truth as civic infrastructure.

A federal-first system for turning rulemaking notices into reliable, source-backed participation-window objects: clear deadlines, provenance, confidence states, and subscriptions.

Architecture thesis

Ingest authoritative sources, preserve raw observations, normalize them into versioned ParticipationWindow records, route uncertainty to review, then publish APIs and feeds downstream products can trust.

What the system solves

DocketClock turns scattered rulemaking notices into durable, subscribable deadline objects. The hard problem is not “show me rules”; it is “tell me what action window exists, when it closes, whether it changed, where to act, and how confident we are.”

Core object

ParticipationWindow is the unit of trust: docket ID, title, agency, window type, open/close times, timezone, submission URL, provenance, confidence, status, and change history.

1. Source adapters

  • Federal Register API
  • Regulations.gov v4
  • Mirrulations / mirror history
  • GovInfo/eCFR adjuncts later

2. Event log

Every observation becomes immutable: source payload hash, fetched-at time, parser version, and extracted candidates. This makes deadline changes auditable instead of mysterious.

3. Normalizer

Entity resolution ties notices, dockets, agencies, CFR parts, comment URLs, and related corrections/reopenings into one canonical window graph.

4. Confidence engine

Rules classify windows as high, medium, low, stale, or conflicting based on source agreement, date parse certainty, freshness, and status semantics.

5. Human review

Only uncertain or high-impact cases need operators: conflicting deadlines, changed close dates, ambiguous timezone/date text, reopened dockets, and user corrections.

6. Registry + API

Postgres stores canonical objects; search indexes support discovery; delivery workers emit webhooks, RSS, ICS, CSV, and bulk snapshots.

Trust model

The product should show uncertainty rather than hiding it. “Unknown,” “stale,” and “conflicting” are first-class states. A bad deadline is worse than no alert, so alerts should be suppressed or downgraded when confidence drops.

Expansion rule

Do not expand to state/local sources until a funded source cluster exists. State/local scraping is where civic infrastructure goes to die unless there is a payer and review loop.
